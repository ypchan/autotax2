# AutoTax2 文件格式说明

## Reference manifest

`prepare-silva` 生成 `autotax2_ref_manifest.tsv`：

```text
key	path
silva_fasta	refdatabases/SILVA_138.2_SSURef_NR99_tax_silva.fasta
silva_sintax_fasta	refdatabases/SILVA_138.2_SSURef_NR99_tax_silva_sintax.fasta
silva_typestrains_fasta	refdatabases/SILVA_138.2_SSURef_NR99_tax_silva_typestrains.fasta
silva_metadata	refdatabases/SILVA_138.2_SSURef.full_metadata
typestrains_accession_ids	refdatabases/typestrains_accessionIDs.txt
silva_taxonomy_tsv	refdatabases/silva_taxonomy.tsv
```

后续 `run` 和 `classify` 会读取其中的路径。

## SINTAX FASTA

AutoTax2 生成的 SINTAX FASTA header 格式：

```text
>seqid;tax=d:Bacteria,p:Firmicutes,c:Bacilli,o:Lactobacillales,f:Lactobacillaceae,g:Lactobacillus,s:Lactobacillus acidophilus;
```

## VSEARCH userout

AutoTax2 最近邻搜索默认字段：

```text
query target id alnlen mism opens qlo qhi tlo thi qcov tcov
```


## assignments.tsv

`autotax2 assign` 生成：

```text
query	cluster_id	assignment_type	identity	qcov	tcov	source
new_seq_1	old_cluster_5	old_cluster	98.7	1.0	1.0	vsearch_usearch_global
new_seq_2	otu97_new_1	new_cluster_seed	100.0			vsearch_cluster
new_seq_3	otu97_new_1	new_cluster_member	97.8			vsearch_cluster
```

`assignment_type` 说明：

- `old_cluster`：新序列达到阈值并分配到已有 centroid。
- `new_cluster_seed`：未匹配序列重新聚类后成为新簇 seed。
- `new_cluster_member`：未匹配序列重新聚类后归入某个新簇。

## sequence_rank_assignment.tsv

Generated by `autotax2 insert-backbone`. Columns include sequence_id, source_label, rank, taxon_id, assignment_type, silva_identity, is_novel, and has_intron.

## SILVA-backed overlap tables

`overlap-backbone` outputs taxon_presence_by_source.tsv, taxon_count_by_source.tsv, source_pairwise_overlap_by_rank.tsv, and source_unique_taxa_by_rank.tsv.


## detect-intron outputs

`analysis_sequences.fa`:

- intron-free analysis sequences for rescued intron-bearing records.
- original sequences for all other records.
- intended as `insert-backbone --input`.

`sequence_version_map.tsv`:

```text
sequence_id	analysis_sequence_id	source_label	has_intron	original_length	analysis_length	centroid_sequence_allowed
seq001	seq001|intron_free	hifimeta	yes	1680	1490	original_only
```

`intron_summary.tsv`:

```text
sequence_id	source_label	subject_id	original_identity	original_length	analysis_sequence_id	analysis_length	has_intron	n_introns	intron_positions_query	intron_lengths	rescued_identity	status
seq001	hifimeta	AB123	91.2	1680	seq001|intron_free	1490	yes	1	720-909	190	0.989100	intron_rescued
```

`intron_regions.bed` is 0-based half-open BED.
